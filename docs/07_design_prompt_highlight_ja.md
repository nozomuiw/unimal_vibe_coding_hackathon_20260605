# Claude Design プロンプト — パターンB：見せ場集中（3ハイライト）

> **目的**: 以下のプロンプトを Claude（Artifacts / design モード）に貼り付け、**短時間でインパクトのあるプレゼンデモ**を制作する。見せ場3つ（裁判官ルーレット、法廷観戦、判決カード）に集中。UI文言は**日本語**。指示書本文も**日本語**（英語版は `07_design_prompt_highlight_en.md`）。
>
> **推奨ワークフロー**: **段階プロンプト（Step 0→5）**を同一スレッドで順に貼る。時間がない場合のみ**一発版**を使用。

---

## このドキュメントの使い方

1. Claude Artifacts の新規会話を開始。
2. マスター仕様を**pin**。
3. **Step 0 → 5** を順に貼る。
4. 裁判官イラストは `JUDGE_IMAGES` に後から差し込む（Step 4）。
5. **3〜5分のライブプレゼン**向け — 聞き取り・受け渡しの詳細は省略。

---

## 共有ブロック（最初に pin）

### マスター仕様（会話先頭に pin）

```
React 単一ファイルコンポーネントのアーティファクトを作成：「家庭内裁判所」の**短編ハイライトデモ** — 裁判官ルーレット、法廷観戦、判決カードの3見せ場に集中。最小限のスプラッシュ（事件名のみ）で開始。LLM/API なし。台本ハードコード。モバイルファースト、日本語 UI、ポップトーン。useState のみ（localStorage 不可）。Tailwind + lucide-react。裁判官画像は JUDGE_IMAGES マップ。全画面に免責表示。
```

### プロダクトコンテキスト

「家庭内裁判所」は家庭内紛争をエンタメ性のある AI 裁判に変えるサービス。ランダムなキャラ裁判官、双方の代理人、必ず判決で終わる。本ハイライトデモはその体験の**感情のピーク**を5分以内で見せる。

### ビジュアル方針

**トーン**: ポップエネルギー最大 — ガチャ演出、ビジュアルノベルチャット、祝祭的な判決カード。

**ベンチマーク**: ガチャ／ルーレットアプリ ＋ キャラチャットゲーム ＋ Duolingo 風カード。

**役割色**: A代理人=シアン、B代理人=マゼンタ、裁判官=琥珀。

**禁止**: 裁判官似顔絵の生成。`JUDGE_IMAGES` またはイニシャルプレースホルダーのみ。

### アーティファクトサンドボックス制約

| ルール | 内容 |
|--------|------|
| 種別 | React 単一ファイルコンポーネント |
| 状態 | useState/useReducer — localStorage 不可 |
| ネットワーク | API/fetch 不可 |
| ライブラリ | Tailwind、lucide-react のみ |
| 画像 | JUDGE_IMAGES マップ — ローカルパス不可、AI 似顔絵禁止 |

### 台本シナリオデータ（ハイライト用サブセット）

パターンAと同一事件:

- **当事者**: 美咲 vs 健太
- **事件名**: 令和7年（家）第1号 夫婦合算家計における自作キーボード無断購入事件
- **一行コンテキスト**（スプラッシュ）: 「夫が相談なしに10万円のキーボードを購入した件」
- **法廷**: CLI ログの5発話すべて（裁判官争点整理 → A弁論 → B弁論 → A再反論 → B再々反論）— **全文 verbatim**
- **判決**: 美咲 75 : 健太 25、判決全文＋提言5項目

全文参照: `docs/05_cli_prototype_log.txt` 207〜473行。

### 本パターンの画面構成

| # | 画面 | 目的 |
|---|------|------|

## 段階プロンプト（メイン）

### Step 0: キックオフ

```
pin したマスター仕様にもとづき、「家庭内裁判所」の**短編ハイライト**プロトタイプを作りたい — 見せ場3つ（ルーレット、法廷、判決）＋最小スプラッシュのみ。

コードを書く前に：
1. 台本どおりの React アーティファクト（API/localStorage なし）であることを確認。
2. 4画面（SPLASH → JUDGE_SELECT → TRIAL → VERDICT）を一覧。
3. 必要なら最大1問だけ質問。

まだコードは書かない。「go」まで待って。
```

### Step 1: 骨組み

```
Step 1 — ハイライト用骨組み。

React 単一ファイル：
- フェーズ: SPLASH → JUDGE_SELECT → TRIAL → VERDICT
- モバイルファースト（~430px）、ポップデザイン、役割色
- フッター免責を常時表示
- 各画面：プレースホルダー＋「次へ」または自動進行フック
- 先頭に TOC コメント

**ユーザーフロー:**
- SPLASH:「裁判を始める」→ JUDGE_SELECT
- JUDGE_SELECT:「ルーレット開始」→ 回転 → TRIAL
- TRIAL:「次へ」または自動で発話 → VERDICT
- VERDICT: タイプライター＋カード、終了

プレースホルダーのみ。useState。シナリオ文はまだ入れない。
```

### Step 2: 中身

```
Step 2 — ハイライト用コンテンツ接続。

骨組みは維持。HIGHLIGHT_SCENARIO を追加（法廷5発話・判決全文を CLI ログから verbatim）。

**SPLASH**: 事件名を演出表示＋「夫が相談なしに10万円の自作キーボードを購入した件」＋ 美咲 vs 健太 ＋「裁判を始める」

**JUDGE_SELECT**: 5人ルーレット → 巌谷 鉄五郎で停止 → 大きな肖像スロット＋「昭和から来た男」＋決め台詞＋開廷宣言（要約可）

**TRIAL**: ヘッダー「代理戦争 — ここからは観戦です」。発話者アバター・名前・役割色付きで5発話を順次表示。全文。

**VERDICT**: 巌谷の判決全文 → サマリーカード（75:25 バー、裁判官名、提言5枚、決め台詞フッター）

聞き取り画面なし。純粋な再生のみ。
```

### Step 3: 演出

```
Step 3 — プレゼンインパクト最大化。コンテンツは変更しない。

- SPLASH: 事件名のスタンプアニメ（拡大＋フェード）、小槌アイコンのバウンス
- ルーレット: 2秒スロット回転、ブラー、巌谷停止時にキラキラ／紙吹雪、決め台詞のポップイン
- 法廷: 2.5秒ごと自動進行（一時停止トグル付き）、話者側からスライドイン、法廷風背景
- 判決: タイプライター（スキップ付き）、割合バー 0 からアニメ、提言カードのフリップ／順次表示、最後に「閉廷」スタンプ

判決に「スキップ」ボタン。ゲームショー的なテンション。
```

### Step 4: 画像スロット

```
Step 4 — JUDGE_IMAGES 配線。パターンAと同様。

<JudgeAvatar> で URL または漢字フォールバック。ルーレットと判決ヘッダーで大きく表示。似顔絵は生成しない。画像 URL を貼る行を教えてください。
```

### Step 5: QA

```
Step 5 — 3〜5分ライブデモ用 QA。

□ SPLASH → VERDICT が20タップ以内（または自動進行で3分）
□ ルーレットは必ず巌谷で停止
□ 法廷・判決の日本語が全文入っている
□ 75:25 バーが正しくアニメ
□ 免責が常時表示
□ 375px モバイル、localStorage/fetch なし
□ JUDGE_IMAGES の null と URL の両方

先頭に「5分プレゼン操作ガイド」コメントを追加。
```
